

# Portfolio Website Plan

Inspired by [omareletr.com](https://www.omareletr.com/), this will be a personal portfolio website with all the same sections, a backend for dynamic content, and a light/dark mode toggle.

---

## 1. Homepage (Hero Section)
- **Live clock** displaying the current time in a monospaced font at the top
- **Your name** in large bold typography with rotating role titles (e.g., "Product Manager.", "YouTuber.", "Podcast Host.")
- **Social media icons** row (LinkedIn, YouTube, Instagram, Medium, Twitter, Goodreads)
- **Location line** with a scrambled/animated text effect (like the reference)
- **Hero photo** on the right side of the hero section
- **Updates timeline** below the hero — a vertical list of date + description + category tag entries

## 2. Navigation
- Sticky top navigation with numbered links: Home (1), Work (2), Blog (3), Podcast (4), Reads (5), About (6), Impossible List (7)
- Clean, minimal styling with keyboard shortcut numbers
- Responsive hamburger menu on mobile

## 3. Work Page
- Showcase of professional experience and projects
- Cards or timeline layout with role, company, description, and date range

## 4. Blog Page
- List of blog posts fetched from the backend (Lovable Cloud database)
- Each post has a title, date, excerpt, and category tag
- Individual blog post page with full content

## 5. Podcast Page
- List of podcast episodes with title, description, and embedded audio/links
- Episode data stored in the backend

## 6. Reads Page
- Book recommendations with cover images, titles, authors, and short reviews
- Grid or list layout

## 7. About Page
- Bio section with personal story and background
- Photo(s) and extended description

## 8. Impossible List Page
- Categorized goals/achievements list (inspired by the "impossible list" concept)
- Items marked as completed or in-progress

## 9. Design & Theme
- **Light mode** as default with clean white backgrounds and subtle gray accents
- **Dark mode** with a toggle switch in the navigation
- Monospaced font accents for time/location, clean sans-serif for body text
- Generous whitespace and minimal aesthetic matching the reference

## 10. Backend (Lovable Cloud)
- **Database tables** for: Blog Posts, Podcast Episodes, Reads, Updates (timeline entries)
- This enables you to add/edit content without touching code
